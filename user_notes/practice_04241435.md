## 1. nginx で LINE Webhook `/callback` がどこを通っているか確認する
### 目的
- LINE Developers Console では `/callback` が設定されているが、確認済みの `lsapp.conf` には `/callback` の `location` が見当たらない。
- 実際に nginx の別ファイルで処理しているのか、設定漏れなのか、運用で使っていないのかを切り分ける。

### 手順
```bash
sudo grep -R "location .*callback" /etc/nginx
sudo grep -R "/callback" /etc/nginx
sudo grep -R "proxy_pass" /etc/nginx
sudo nginx -T | less
```

### 見るポイント
- `/callback` を扱う `location` が存在するか
- もし存在するなら、どの conf ファイルにあるか
- `/callback` が `127.0.0.1:8080` に proxy されているか
- 何も出ない場合、LINE Webhook は現行 nginx 設定では通らない可能性がある
